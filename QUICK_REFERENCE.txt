WHAT IS A STUB?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

A stub is any function with NO real implementation:
  • Empty body: void Func() { }
  • Just guard/unguard: guard("x"); unguard;
  • Just return: return 0;
  • Just TODO: // TODO: implement this
  • Just error: appUnimplemented();

These 1,179 stubs must be implemented for the game to work properly.

COMMON QUESTIONS

Q: Why so many stubs?
A: Decompilation from retail binary was incomplete. Stubs are placeholders
   that need implementation based on decompiled code and documentation.

Q: What's a DIVERGENCE comment?
A: Marks intentional differences from retail. Finding 34 stubs with this
   marker suggests cleanup was incomplete.

Q: Should we implement all stubs?
A: Yes. Each stub is a gap in functionality. Prioritize by tier.

Q: Can we skip some stubs?
A: Not recommended. Each stub impacts functionality. Some (audio) are
   critical; others (editor tools) can be deferred.

Q: How do we know if an implementation is correct?
A: Compare against retail behavior. Document deviations from retail.
   Use test suite to validate.
